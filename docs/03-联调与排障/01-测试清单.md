# 测试清单

这份文档帮助你在正式联调前快速确认环境是否正确。

## 一、联调前检查

### 基础环境

- [ ] 已安装 `rn-alipay`
- [ ] 已重新编译 Android
- [ ] 使用 Android 真机测试
- [ ] 真机已安装支付宝 App
- [ ] 宿主 App 可以联网
- [ ] 服务端能正确返回 `orderInfo`
- [ ] 服务端 `notify_url` 可公网访问

### 构建环境

- [ ] `google()` 可访问
- [ ] `mavenCentral()` 可访问
- [ ] Android 构建未被代理或镜像阻断
- [ ] 宿主工程 `minSdkVersion >= 21`

## 二、推荐测试顺序

### 第一步：确认库已正确链接

```ts
import Alipay from 'rn-alipay';

const installed = await Alipay.isAlipayInstalled();
const version = await Alipay.getVersion();
console.log({ installed, version });
```

如果这一步就报错，先不要进入支付联调。

### 第二步：确认服务端确实返回了 `orderInfo`

重点检查：

- 是否是字符串
- 是否非空
- 是否包含签名字段
- 是否来自服务端，而不是客户端本地拼接

### 第三步：发起真实支付

```ts
const result = await Alipay.pay(orderInfo);
console.log('alipay result:', result);
```

### 第四步：确认服务端最终订单状态

无论客户端结果如何，都应该：

- 等待 `notify_url`
- 或主动查单
- 以服务端最终状态更新 UI

## 三、联调建议顺序

1. 先验证 `isAlipayInstalled()` / `getVersion()`
2. 再验证服务端 `orderInfo`
3. 再做真机支付
4. 最后验证 `notify_url` 与查单逻辑
