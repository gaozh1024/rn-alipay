# 错误码与处理建议

本文档描述 `rn-alipay` 可能抛出的客户端错误，以及建议处理方式。

## 一、JS 层错误

### `orderInfo 必须是非空字符串。`

来源：JS 参数校验。

可能原因：

- 传入了空字符串
- 传入了非字符串
- 服务端返回字段为空或字段名错误

建议：

- 检查服务端返回体
- 确认调用 `pay()` 时传的是 `data.orderInfo`

## 二、模块未链接错误

### `The package 'RnAlipay' doesn't seem to be linked...`

说明：

- Android 原生模块未被正确编译进宿主 App

建议：

- 重新执行 Android 构建
- 确认 React Native 自动链接正常
- 必要时清理缓存后重编

## 三、Android 原生错误码

### `E_INVALID_ORDER_INFO`

说明：Native 层收到空 `orderInfo`。

### `E_NO_ACTIVITY`

说明：当前没有可用前台 Activity。

建议：

- 在页面可见后再发起支付
- 不要在 App 启动瞬间自动触发支付

### `E_PAY_IN_PROGRESS`

说明：已有一笔支付流程正在执行。

建议：

- 前端按钮加 loading / 防抖
- 一次只允许发起一笔支付

### `E_PAY_FAILED`

说明：调用支付宝 SDK 过程中抛出异常。

建议：

- 记录异常 message
- 检查支付宝客户端安装情况
- 检查 `orderInfo` 是否完整有效
- 检查当前设备环境

### `E_GET_VERSION_FAILED`

说明：获取 SDK 版本失败。

### `E_PACKAGE_CHECK_FAILED`

说明：查询支付宝是否安装时发生异常。

## 四、支付宝支付结果状态码

这些不是本库抛出的异常，而是支付结果状态：

- `9000`：成功
- `8000`：确认中
- `4000`：失败
- `5000`：重复请求
- `6001`：用户取消
- `6002`：网络错误
- `6004`：未知

## 五、推荐处理模板

```ts
try {
  const result = await Alipay.pay(orderInfo);

  if (result.succeeded) {
    // 去服务端确认
  } else if (result.pending) {
    // 轮询服务端
  } else if (result.cancelled) {
    // 用户取消
  } else {
    // 失败提示
  }
} catch (error) {
  const message = error instanceof Error ? error.message : '未知错误';
  console.error('alipay pay failed:', message);
}
```
