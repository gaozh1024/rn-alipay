# 宿主应用接入

这份文档面向 React Native 宿主应用开发者。

## 一、前置条件

建议宿主项目满足：

- React Native `0.72+`
- Android `minSdk >= 21`
- Android 真机测试
- 可访问 `google()` 和 `mavenCentral()`
- 已有服务端接口可返回签名后的 `orderInfo`

## 二、安装依赖

```bash
pnpm add rn-alipay
# 或 npm install rn-alipay
```

重新编译：

```bash
npx react-native run-android
```

> 原生库接入后不要只热更新 JS，必须重新编译 Android。

## 三、自动链接说明

当前库已提供 React Native Android 自动链接配置。

正常情况下，宿主项目**不需要手动修改**：

- `MainApplication`
- `settings.gradle`
- `app/build.gradle`

只有在较老的 React Native 项目里，才可能需要手动 link。

## 四、宿主 Android 构建要求

### 1. 仓库源

```gradle
repositories {
  google()
  mavenCentral()
}
```

### 2. 推荐版本

- `compileSdkVersion >= 34`
- `targetSdkVersion >= 34`
- `minSdkVersion >= 21`
- Kotlin `1.9.x`
- Java `11`

### 3. 默认依赖的支付宝 SDK

```gradle
implementation "com.alipay.sdk:alipaysdk-android:15.8.40"
```

如需覆盖版本，可在宿主 `android/build.gradle` 中配置：

```gradle
ext {
  alipaySdkVersion = "15.8.40"
}
```

## 五、客户端接入示例

### 1. 调起前先判断是否安装支付宝

```ts
const installed = await Alipay.isAlipayInstalled();
if (!installed) {
  throw new Error('设备未安装支付宝客户端');
}
```

### 2. 可选：记录 SDK 版本

```ts
const version = await Alipay.getVersion();
console.log('alipay sdk version:', version);
```

### 3. 发起支付

```ts
const result = await Alipay.pay(orderInfo, { showLoading: true });
```

## 六、支付后的推荐处理方式

客户端返回值只能用于引导 UI，不要直接作为最终入账依据。

建议：

- `9000`：提示“支付结果处理中”，并立刻向服务端查单
- `8000` / `6004`：提示“支付确认中”，轮询服务端订单状态
- `6001`：提示“用户取消”
- 其他：提示失败并允许重试

## 七、错误用法

不要这样做：

- 在客户端本地签名
- 把支付宝私钥放进 App
- 收到 `9000` 就直接发货
- 依赖客户端同步回调做最终订单状态
