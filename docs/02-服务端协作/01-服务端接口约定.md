# 服务端接口约定

本文档定义 React Native 客户端与业务服务端之间围绕支付宝支付的最小接口契约。

## 一、设计原则

- 客户端不拼签名
- 客户端不持有支付宝私钥
- 服务端返回完整 `orderInfo`
- 服务端提供最终订单状态查询接口

## 二、创建支付订单

### 请求

`POST /pay/alipay/order`

### 请求体示例

```json
{
  "amount": "0.01",
  "subject": "测试商品",
  "outTradeNo": "order_202603250001",
  "body": "可选：商品描述",
  "passbackParams": "可选：业务透传参数"
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "orderInfo": "app_id=...&biz_content=...&sign=...",
    "outTradeNo": "order_202603250001",
    "expireTime": "2026-03-25T17:30:00+08:00"
  }
}
```

### 字段要求

- `orderInfo`：必须是可直接传给 `Alipay.pay()` 的完整字符串
- `outTradeNo`：客户端后续查单要用
- `expireTime`：可选，用于前端展示倒计时

## 三、查询订单状态

### 请求

`GET /pay/orders/:outTradeNo`

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "outTradeNo": "order_202603250001",
    "status": "SUCCESS",
    "paidAt": "2026-03-25T17:05:00+08:00"
  }
}
```

### 推荐状态值

- `CREATED`
- `PAYING`
- `PENDING`
- `SUCCESS`
- `FAILED`
- `CLOSED`

## 四、客户端推荐处理逻辑

### `9000`

- 提示“支付结果处理中”
- 立即查服务端订单状态
- 未确认时继续轮询

### `8000` / `6004`

- 提示“支付确认中”
- 按 `outTradeNo` 轮询服务端状态

### `6001`

- 视为用户取消
- 保持订单待支付
- 提供重新支付入口

### 其他失败状态

- 展示错误提示
- 保持订单未完成
- 允许重试

## 五、不建议的接口设计

不要让服务端返回：

- 一堆待拼接字段让 App 自己签名
- 商户私钥
- 未落库的“最终成功态”
