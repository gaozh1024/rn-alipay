# 支付链路与时序

本库只负责 **客户端拉起支付宝支付**。

完整支付链路必须由服务端参与完成。

## 一、服务端职责

服务端至少需要负责：

- 生成商户订单号 `out_trade_no`
- 组装支付宝下单参数
- 使用商户私钥签名
- 生成完整 `orderInfo`
- 返回给 App
- 接收 `notify_url` 异步通知
- 验签并更新订单状态
- 提供订单查询接口给 App 主动轮询

## 二、推荐链路

```text
RN App
  └─ 请求服务端创建支付宝订单
      └─ 服务端组装参数并签名
          └─ 返回 orderInfo 给 App
              └─ App 调用 rn-alipay.pay(orderInfo)
                  └─ 支付宝 App / SDK 完成支付
                      ├─ 客户端收到同步结果
                      └─ 支付宝服务器异步通知 notify_url
                           └─ 服务端验签并更新订单状态
```

## 三、推荐时序图

```text
App                业务服务端                支付宝
 |                     |                        |
 | 创建订单            |                        |
 |-------------------->|                        |
 |                     | 组装参数并签名         |
 |                     |----------------------->| （概念示意）
 |                     |<-----------------------|
 | 返回 orderInfo      |                        |
 |<--------------------|                        |
 | pay(orderInfo)      |                        |
 |--------------------------------------------->|
 | 同步结果回调        |                        |
 |<---------------------------------------------|
 | 查单                |                        |
 |-------------------->|                        |
 |                     | 等待 notify / 主动查单 |
 |                     |<---------------------->|
 | 最终状态            |                        |
 |<--------------------|                        |
```

## 四、为什么不能在客户端签名

因为客户端代码可以被反编译，商户私钥放在 App 中会直接泄露。

所以：

- 私钥只能放在服务端
- 签名只能由服务端执行
- 最终订单状态只能以服务端为准

## 五、notify_url 处理原则

服务端收到通知后应：

1. 验签
2. 校验 `out_trade_no`
3. 校验订单金额
4. 校验商户身份
5. 幂等更新订单状态
6. 记录通知原文与处理日志

## 六、最重要的一条

**客户端负责发起支付，服务端负责确认支付。**
